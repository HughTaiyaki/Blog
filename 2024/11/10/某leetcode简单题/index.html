static constexpr

static 是在类中用于指定该变量是静态的，就是说变量在整个类内是共享的，而不是属于任何一个具体的对象。静态成员变量只有一份存储空间；也就是无论创几个对象，所有对象都用这一个内存。

那么 constexpr 是什么呢？我查了一下，发现它是 C++11 引入的关键字，用于指定一个常量表达式，在编译的时候就可以计算出结果的值，那就可以优化性能。那么理所当然的 contexpr 变量必须是常量，要在声明的时候就定义。

那么为什么要 static 呢，将常量声明为static 之后，常量就变成一个类成员，类可以直接访问了（因为 leetcode 都是让你实现一个类）。其实不加也可以跑，dirs 本来就是一个常量数组，但是如果创建了很多个 NeighborSum 对象，那它们都会有一个 dirs 的独立拷贝，很划不来，太浪费了！

LRXYvalues 与 移动语义

左值 右值

移动语义是 CPP 中很重要的一个概念，搞懂了移动语义，很多 cpp 问题就迎刃而解了。

  • 左值(Lvalue)

左值指的是可以出现在赋值语句左边的对象。是指向某个内存位置的对象或变量，可以被修改或者赋值。（但有时候编译器会直接对其进行优化，以避免冗余的内存分配和复制操作）

因此，我们在后面的讨论中，始终从概念层面上认为 lvalue 会占用内存。编译器将这个内存优化掉是实践层面的事情，如前所述，并不会影响我们这样讨论的正确性。

左值可以持久存在（如变量），并且你可以获取它的地址

  • 右值(Rvalue)

右值指的是不能出现在赋值语句左边的临时值，没有固定的位置，也不持久存在。

右值通常表示表达式计算后的临时结果，或者可以被“移动”的数据。

移动语义

在C++11引入的移动语义，目的是为了提高性能，尤其是在处理大数据结构（比如容器类）时避免不必要的深拷贝。

移动语义如何工作？

移动语义主要通过“右值引用”（&&）来实现。右值引用允许你捕获临时对象（右值）并将其资源“移动”到另一个对象中，而不是拷贝资源。

如果没有移动语义，std::vector<int>的返回值将会进行深拷贝；如果使用了移动语义，返回的对象会通过移动构造函数将内存的所有权转移，而不是拷贝数据。

右值饮用

  • T&& 语法表示右值引用，它是一个引用，但只能绑定到右值上。
  • 右值引用允许你通过“移动”资源来优化程序性能。

首先，了解为什么需要移动构造函数和移动赋值运算符。

深拷贝（deep copy）会将一个对象的所有数据复制到另一个对象。如果对象很大（例如，包含大量动态分配内存或数据），深拷贝会导致性能上的瓶颈，特别是当对象作为函数参数传递或从函数返回时。移动操作的目标是避免不必要的深拷贝。移动语义允许我们“转移”资源的所有权，而不是复制数据，从而提高效率。例如，std::vector 是一个容器类，它内部维护动态分配的内存。如果你直接复制一个 std::vector 对象，所有的数据都会被复制一份。但如果你使用“移动”，则不需要复制数据，只需转移内存的所有权。
